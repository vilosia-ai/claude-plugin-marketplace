# Create Testcases

## Feature File: $ARGUMENTS
Analyse the given fetaure file and create up to 10 testcases with testdata that cover all edge cases for later implementation.
Create one new file with all test cases in location `.vilosia/features/{feature-name-tests}.md`

The Testcases must excatly follow the following format. Do not create testcode.

- Testcase description
- Endpoint Input
- Endpoint Output
